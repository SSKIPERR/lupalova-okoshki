# Лупалова — бот «Свободные окошки»

Каждый день автоматически (даже если ваш компьютер выключен):
1. в **9:00 МСК** бот пишет в Telegram-группу: «Доброе утро, жду свободные окошки на сегодня, пришлите имена + какая услуга»
2. в **9:30 МСК** бот читает ответы мастеров в этой же группе, собирает сторис по фирменному стилю и присылает картинку 1080×1920 туда же в группу — готовую к публикации в Instagram Stories.

Работает на GitHub Actions (бесплатно), без необходимости держать компьютер включённым.

## Что нужно сделать один раз

### 1. Создать репозиторий
На github.com → New repository → название, например `lupalova-okoshki` → **Private** → Create.

### 2. Загрузить файлы
Загрузите все файлы и папки из этого архива в репозиторий, **сохраняя пути** (`.github/workflows/...`, `scripts/...`, `fonts/...`, `requirements.txt`, `state/...`). Через веб-интерфейс GitHub это можно сделать через «Add file → Upload files», перетащив всю папку целиком — GitHub сохранит структуру вложенных папок.

### 3. Включить права на запись для Actions
Settings → Actions → General → раздел **Workflow permissions** → выбрать **«Read and write permissions»** → Save.
(Это нужно, чтобы бот мог сам сохранять служебное состояние между запусками.)

### 4. Получить токен бота и chat_id группы
1. В Telegram напишите **@BotFather** → `/newbot` → задайте имя → получите **токен** (выглядит как `123456:ABC-DEF...`).
2. Добавьте бота в вашу группу (как обычного участника).
3. Напишите в группе любое сообщение.
4. Откройте в браузере: `https://api.telegram.org/bot<ТОКЕН>/getUpdates` (вставьте свой токен).
5. В ответе найдите `"chat":{"id": -100xxxxxxxxx, ...}` — это **chat_id** группы (число, обычно отрицательное).

### 5. Добавить секреты в репозиторий
Settings → Secrets and variables → Actions → **New repository secret**:
- `TELEGRAM_BOT_TOKEN` — токен из шага 4
- `TELEGRAM_CHAT_ID` — chat_id из шага 4

### 6. Проверить вручную
Actions → выбрать workflow **«Утренний вопрос про окошки»** → **Run workflow** → проверить, что сообщение пришло в группу.
Затем написать в группу тестовый ответ, например: `Оля 12:00, 15:00, Галя 14:00`.
Запустить вручную workflow **«Сбор ответов и сторис»** → проверить, что в группу пришла готовая картинка.

После успешного теста всё работает само, по расписанию.

## Важно знать

- Расписание GitHub Actions может опаздывать на несколько минут в часы пиковой нагрузки — это нормальная особенность бесплатного тарифа, точность ±5-10 минут.
- Если в репозитории совсем не будет активности 60 дней, GitHub автоматически отключает cron — но бот сам коммитит файлы состояния каждый день, так что это не страшно.
- Бот узнаёт мастеров по имени: Вера, Оля/Ольга, Яна, Ирина, Ксения, Галя/Галина (список — в `scripts/collect_and_render.py`, переменная `MASTERS`). Время ищет в формате `12:00` или `12:00 - 15:00` сразу после имени в том же сообщении. Если разбор работает не идеально на реальных ответах — пришлите примеры, поправим регулярку.
- Если на момент сбора (9:30) ответов ещё нет, бот напишет в группу «Пока нет ответов...» вместо картинки — это нормально, можно запустить «Сбор ответов и сторис» вручную позже, когда ответы появятся.
- Поменять время: в файлах `.github/workflows/*.yml` строка `cron: "M H * * *"` — час указывается в UTC (МСК = UTC+3, без перевода времени).
